How to Use Aqua Daily Briefing

The Daily Briefing system combines your Outlook Data (Emails & Meetings) with your Daily Priorities into a single, easy-to-read report.

The Two-Part System

The system consists of two parts working together:

  1. The Web Dashboard: Where you manage your tasks, project updates, and manage security.
  2. The Outlook Agent: A small program on your PC that automatically grabs latest emails and calendar events.

Step 1: Manage Content & Security (Web Dashboard)

Add/edit your projects/tasks/daily tasks and set your security preferences.

  1. Go to the Admin Dashboard and log in.
  2. Security: Navigate to the **Settings** tab to set a passcode for your secure Daily Briefing link (?daily=).
  3. Daily Tasks: Add your top 3-5 priorities for the day.
  4. Projects: Update the status (e.g., "On Track", "Delayed") of your active projects.

Tip: You only need to set the passcode once, but you can change it anytime in the Settings tab.

Step 2: Sync Outlook (The Agent)

Since the web cannot see your local Outlook, you must run the "Agent" to sync that data.

  1. Double-click G:\AquaDailyBriefing.exe.
  2. Wait a few seconds for the confirmation popup.

This uploads your Unread Emails (Last 24h) and Upcoming Meetings (Today-Friday) to the report.

Step 3: View Your Report

You have two ways to view your report:

A. Daily Briefing (Secure/Private)

URL: https://dailyaquabriefing.github.io/?daily=YOUR_ID

  • Shows Tasks, Projects, Active Tasks, Meetings, and Emails.
  • **Requires the passcode** you set on the Admin Settings tab.

B. Weekly Report (Public/Shareable)

URL: https://dailyaquabriefing.github.io/?report=YOUR_ID

  • Shows Tasks, Projects, and Active Tasks.
  • Does **NOT** require a passcode and is suitable for sharing.

Pro-Tip: Automate the Outlook Sync

The Outlook Agent can be configured to run automatically at login using the Startup folder.

Run at Login (Recommended)

The agent manages its own startup shortcut based on your config file.

  1. Open G:\dailybriefingconfig.txt.
  2. Set RunAtLogin=Yes to have the agent automatically create a shortcut in your Startup folder every time it runs.
  3. Set RunAtLogin=No to have the agent delete the shortcut if it exists, preventing it from running at login.

Note: If running automatically, you might want to open G:\dailybriefingconfig.txt and change Prompt=Yes to Prompt=No so the agent runs silently in the background without a confirmation box.
